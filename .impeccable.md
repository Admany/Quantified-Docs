## Design Context

### Users

Minecraft mod developers and JVM engineers integrating **Quantified API** into mods and servers. They use these docs while coding: scanning for APIs, copying snippets, and checking behavior. Context is IDE plus browser side-by-side; clarity and scannability matter more than marketing polish.

### Brand Personality

**Precise**, **studio-grade**, **approachable-direct** — authored by BlackRift Studios / Admany without corporate stiffness. Voice in copy can stay informal where it helps; visual layer should stay calm and intentional so legal and API content feel trustworthy.

### Aesthetic Direction

Dark, low-noise canvas with **silver/chrome** accents (not cyan-neon generic tech). Typography and spacing carry hierarchy; motion stays subtle (ambient background only, respectful of reduced-motion). Prefer left-aligned prose, intentional vertical rhythm, and grouped blocks over decorative chrome.

Reference: the existing Quantified API sidebar rail, shimmering title treatments, monospace for code paths.

Avoid: overcrowded grids, ambiguous spacing, cyan-on-purple “AI SaaS” tropes (already toned down).

### Design Principles

1. **Rhythm beats decoration** — consistent spacing scale, tight inside groups / loose between sections.
2. **Scan-first** — headings, tables, and code anchors should pop within two seconds without extra ornaments.
3. **One monospace voice for code** — inline and blocks share a clear mono layer; body stays readable sans.
4. **Restraint in depth** — borders and fills are subtle; elevation only when signaling interactivity.
